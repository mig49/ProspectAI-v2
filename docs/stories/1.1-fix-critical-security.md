## Story

**Como** owner do ProspectAI,
**Quero** que as vulnerabilidades de open redirect e falta de ownership check sejam corrigidas,
**Para que** os usuarios estejam protegidos contra phishing e acesso nao autorizado a dados.

---

## Acceptance Criteria

- [ ] AC1: O parametro `next` em `/auth/callback` so aceita paths internos (whitelist: `/`, `/login`)
- [ ] AC2: URLs externas no param `next` sao ignoradas e redirecionam para `/`
- [ ] AC3: POST `/api/prospect/report` verifica que o usuario autenticado e dono do lead
- [ ] AC4: Requisicoes de report para leads de outros usuarios retornam 403
- [ ] AC5: Testes manuais confirmam que ambas correcoes funcionam

---

## Scope

### In
- Validacao do param `next` em `app/auth/callback/route.ts`
- Auth check + ownership em `app/api/prospect/report/route.ts`

### Out
- Rate limiting (Story 1.2)
- Novas features de auth

## Tasks

- [x] **Task 1:** Corrigir open redirect em `/auth/callback/route.ts`
  - Criar whitelist de paths permitidos (`/`, `/login`)
  - Validar `next` param contra whitelist
  - Default para `/` se path invalido
  - (AC1, AC2)

  - Testar redirect com `?next=https://evil.com` (deve ir para `/`)
  - Testar report de lead de outro usuario (deve dar 403)

## Dev Notes

**SEC-1 — Open Redirect:**
- Arquivo: `app/auth/callback/route.ts:10`
- Atual: `const next = searchParams.get("next") ?? "/"`
- Fix: Validar contra whitelist antes de usar

**SEC-2 — Ownership Check:**
- Arquivo: `app/api/prospect/report/route.ts`
- Atual: Nao verifica user_id do lead
- Fix: Query `leads` table filtrando por `user_id = auth.uid()`
- Nota: Requer refactor para receber `lead.id` ao inves de dados completos

---

## Risk Assessment

| Risco | Mitigacao |
|-------|-----------|
| Ownership check pode quebrar flow existente | Manter backward-compatible: aceitar lead_id OU lead data |
| Whitelist muito restritiva | Comecar com `/` e expandir conforme necessidade |

---

## Testing

- Testar redirect com `?next=https://evil.com` → deve redirecionar para `/`
- Testar redirect com `?next=/` → deve redirecionar para `/`
- Testar redirect com `?next=/login` → deve redirecionar para `/login`
- Testar POST `/api/prospect/report` sem auth → deve retornar 401
- Testar POST `/api/prospect/report` com lead de outro user → deve retornar 403
- Testar POST `/api/prospect/report` com lead proprio → deve funcionar normalmente
