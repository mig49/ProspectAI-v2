## Story

**Como** developer do ProspectAI,
**Quero** ter um framework de testes configurado com testes para os fluxos criticos,
**Para que** eu possa refatorar e adicionar features com confianca, sem regressoes silenciosas.

---

## Acceptance Criteria

- [ ] AC1: Vitest configurado com suporte a TypeScript e path aliases (@/*)
- [ ] AC2: Script `npm test` roda todos os testes
- [ ] AC3: Testes para `lib/validation.ts` — todos schemas testados (happy + error paths)
- [ ] AC4: Testes para `POST /api/prospect` — input validation, error handling, response shape
- [ ] AC5: Testes para `POST /api/prospect/report` — input validation, error handling
- [ ] AC6: Testes para `hooks/use-prospect-search.ts` — search, error, clear
- [ ] AC7: Testes para `hooks/use-lead-report.ts` — load, error, retry
- [ ] AC8: Testes para `lib/supabase/middleware.ts` — auth redirect logic
- [ ] AC9: Cobertura minima de 60% nos arquivos testados
- [ ] AC10: CI-ready: `npm test` retorna exit code 0 quando tudo passa

---

## Scope

### In
- Setup Vitest + testing-library/react
- Testes unitarios para lib/ (validation, utils)
- Testes de API routes (mock Gemini + Supabase)
- Testes de hooks (mock fetch)
- Testes de middleware (mock request/response)

### Out
- Testes E2E (Playwright/Cypress)
- Testes de componentes React visuais
- Testes de performance/load

---

## Tasks

- [ ] **Task 1:** Setup Vitest
  - `npm install -D vitest @testing-library/react @testing-library/jest-dom jsdom`
  - Criar `vitest.config.ts` com path aliases e jsdom environment
  - Adicionar script `"test": "vitest run"` e `"test:watch": "vitest"` ao package.json
  - (AC1, AC2, AC10)

- [ ] **Task 2:** Testes de validacao
  - Criar `lib/__tests__/validation.test.ts`
  - Testar searchParamsSchema: valid input, missing fields, max length exceeded
  - Testar reportParamsSchema: valid input, missing lead, missing service
  - Testar leadSchema: valid lead, defaults, optional fields
  - (AC3)

- [ ] **Task 3:** Testes de API routes
  - Criar `app/api/prospect/__tests__/route.test.ts`
  - Mock `@google/genai` e `@/lib/supabase/server`
  - Testar: 400 (input invalido), 500 (sem API key), 200 (response shape)
  - Criar `app/api/prospect/report/__tests__/route.test.ts`
  - Testar: 400 (input invalido), 200 (retorna report string)
  - (AC4, AC5)

- [ ] **Task 4:** Testes de hooks
  - Criar `hooks/__tests__/use-prospect-search.test.ts`
  - Mock global fetch
  - Testar: search retorna leads, error handling, clearResults
  - Criar `hooks/__tests__/use-lead-report.test.ts`
  - Testar: auto-fetch on mount, error state, retry
  - (AC6, AC7)

- [ ] **Task 5:** Teste de middleware
  - Criar `lib/supabase/__tests__/middleware.test.ts`
  - Mock Supabase client e cookies
  - Testar: authenticated user passa, unauthenticated redireciona
  - Testar: /login e /auth nao redirecionam
  - (AC8)

- [ ] **Task 6:** Verificar cobertura
  - Rodar `vitest run --coverage`
  - Garantir >= 60% nos arquivos testados
  - (AC9)
