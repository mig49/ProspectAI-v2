## Dev Notes

**ESLint:** Ao ativar no build, erros existentes vao aparecer. Corrigir todos antes de dar merge.
**RLS:** Executar SQL diretamente no Supabase Dashboard > SQL Editor. Depois atualizar schema.sql local.
**WITH CHECK:** A clause `WITH CHECK` garante que o user nao pode mudar o `user_id` de um registro para outro usuario durante UPDATE.
