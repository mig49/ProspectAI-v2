## Story

**Como** usuario do ProspectAI,
**Quero** ver feedback claro de progresso durante buscas longas e ter documentacao completa do projeto,
**Para que** eu saiba que o sistema esta funcionando e novos developers possam contribuir rapidamente.

---

## Acceptance Criteria

- [ ] AC1: Durante busca de leads (15-30s), UI exibe steps de progresso (ex: "Consultando Google Maps...", "Analisando leads...", "Calculando scores...")
- [ ] AC2: Durante geracao de report, UI exibe mensagem contextual com tempo estimado
- [ ] AC3: Progress feedback usa Motion para transicoes suaves
- [ ] AC4: README.md atualizado com: descricao, setup, env vars, arquitetura, API docs, deploy
- [ ] AC5: README inclui diagrama de arquitetura (mermaid ou ASCII)
- [ ] AC6: README inclui documentacao dos endpoints de API

---

## Scope

### In
- Progress steps animados durante busca
- Mensagens contextuais durante report
- README completo e profissional
- Docs inline dos endpoints

## Risk Assessment

| Risco | Mitigacao |
|-------|-----------|

| README pode ficar desatualizado rapidamente | Focar em info estavel (setup, env vars, arquitetura) |

## Dev Notes

**Progress steps simulados:** Como o Gemini nao emite eventos de progresso, usamos um timer que avanca os steps automaticamente. Isso da ao usuario a sensacao de que algo esta acontecendo.

```typescript
const PROGRESS_STEPS = [
  { delay: 0, message: "Consultando Google Maps..." },
  { delay: 5000, message: "Analisando presenca digital..." },
  { delay: 12000, message: "Calculando scores..." },
  { delay: 20000, message: "Finalizando..." },
];
```
