## Story

**Como** usuario Pro/Scale do ProspectAI,
**Quero** receber alertas semanais com novos negocios no meu distrito,
**Para que** leads cheguem ate mim automaticamente sem eu precisar buscar.

---

## Acceptance Criteria

- [ ] AC1: Pagina `/settings/alerts` permite configurar ICP, servico e distrito(s) para monitorar
- [ ] AC2: Pro pode configurar 1 alerta (1 distrito), Scale ate 5 alertas
- [ ] AC3: Cron job roda 1x por semana (segunda 8h) via Vercel Cron
- [ ] AC4: Cron executa busca Gemini para cada alerta ativo, compara com leads ja conhecidos
- [ ] AC5: Apenas leads NOVOS (nao existentes na tabela leads do usuario) sao incluidos no alerta
- [ ] AC6: Email enviado ao usuario com resumo: quantidade de novos leads, top 5 por pain score
- [ ] AC7: Email inclui link para ver todos os novos leads no app
- [ ] AC8: Tabelas `alert_configs` e `alert_results` criadas com RLS
- [ ] AC9: Toggle on/off por alerta sem deletar a configuracao

## Technical Notes

### Schema

```sql
create table alert_configs (
  id uuid primary key default gen_random_uuid(),
  user_id uuid references auth.users not null,
  icp text not null,
  service text not null,
  district text not null,
  city text,
  enabled boolean default true,
  created_at timestamptz default now()
);

create table alert_results (
  id uuid primary key default gen_random_uuid(),
  alert_config_id uuid references alert_configs not null,
  ran_at timestamptz default now(),
  new_leads_count int default 0,
  status text check (status in ('success','failed','no_new_leads'))
);
```

### Vercel Cron

```json
// vercel.json
{
  "crons": [{
    "path": "/api/cron/weekly-alerts",
    "schedule": "0 8 * * 1"
  }]
}
```
