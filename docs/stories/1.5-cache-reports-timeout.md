## Story

**Como** usuario do ProspectAI,
**Quero** que reports ja gerados sejam servidos instantaneamente e que buscas nao travem indefinidamente,
**Para que** a experiencia seja rapida e confiavel.

---

## Acceptance Criteria

- [ ] AC1: Reports gerados sao salvos na coluna `detailed_report` da tabela `leads`
- [ ] AC2: Se `detailed_report` ja existe para um lead, e retornado sem chamar Gemini
- [ ] AC3: Botao "Regenerar Relatorio" permite forcar nova geracao
- [ ] AC4: Chamadas Gemini tem timeout de 60 segundos
- [ ] AC5: Timeout resulta em mensagem de erro amigavel (nao trava a UI)
- [ ] AC6: Cache reduz chamadas Gemini em revisitas ao mesmo lead

---

## Scope

### In
- Persistencia de reports na coluna `detailed_report` (ja existe no schema)
- Check de cache antes de chamar Gemini
- AbortController com timeout de 60s
- Botao de regenerar no frontend

### Out
- TTL/expiracao de cache (pode ser v2)
- Cache em Redis/Edge
- Streaming de responses

---

## Tasks

- [ ] **Task 1:** Implementar cache de reports no backend
  - Em `/api/prospect/report/route.ts`:
  - Receber `lead_id` no request body
  - Query Supabase: `leads.select('detailed_report').eq('id', lead_id).single()`
  - Se `detailed_report` nao null E nao `force_regenerate`, retornar cached
  - Se null ou force, gerar via Gemini e salvar com `leads.update({ detailed_report })`
  - (AC1, AC2)

- [ ] **Task 2:** Adicionar timeout nas chamadas Gemini
  - Em ambos endpoints (`/api/prospect` e `/api/prospect/report`):
  - Usar `AbortController` com `setTimeout(60000)`
  - Wrap chamada Gemini com abort signal
  - Catch `AbortError` e retornar 504 com mensagem
  - (AC4, AC5)

- [ ] **Task 3:** Botao regenerar no frontend
  - Em `LeadDetail.tsx`: adicionar botao "Regenerar" ao lado de "Copiar"
  - Passar `force_regenerate: true` no body do request
  - Em `hooks/use-lead-report.ts`: aceitar param `forceRegenerate`
  - (AC3)

- [ ] **Task 4:** Atualizar hook para suportar cache
  - `use-lead-report.ts`: passar `lead_id` no request body
  - Manter backward-compatible com flow existente
  - (AC6)

## Dev Notes

**Coluna `detailed_report`:** Ja existe no schema (`detailed_report TEXT`), esta pronta para uso.

**Timeout pattern:**
```typescript
const controller = new AbortController();
const timeout = setTimeout(() => controller.abort(), 60000);
try {
  const result = await genAI.models.generateContent({ ...config, signal: controller.signal });
} finally {
  clearTimeout(timeout);
}
```

**Nota:** O SDK @google/genai pode nao suportar AbortSignal diretamente. Alternativa: `Promise.race` com timeout promise.
