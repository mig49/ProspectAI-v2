## Story

**Como** usuario do ProspectAI,
**Quero** gerenciar meus leads salvos com status e notas,
**Para que** eu possa acompanhar cada negociacao e nao perder oportunidades.

---

## Acceptance Criteria

- [ ] AC1: Pagina `/leads` lista todos os leads salvos do usuario com filtro por status
- [ ] AC2: Status por lead: novo, contatado, em negociacao, ganho, perdido — alteravel via dropdown
- [ ] AC3: Campo de notas livres por lead (textarea, salva automaticamente com debounce)
- [ ] AC4: Campo `contacted_at` preenchido automaticamente quando status muda para "contatado"
- [ ] AC5: Dashboard no topo da pagina: total leads, contatados, em negociacao, ganhos, perdidos, taxa conversao
- [ ] AC6: Usuarios Free veem apenas os ultimos 5 leads salvos, com upsell para ver todos
- [ ] AC7: Busca por nome do lead na lista
- [ ] AC8: Ordenacao por data, status ou digital pain score

---

## Scope

### In
- Pagina `/leads` com lista + filtros + dashboard
- Alteracao da tabela `leads` (add status, notes, contacted_at)
- API routes para CRUD de status/notas
- Dashboard com metricas simples
- Componentes: LeadRow, LeadFilters, LeadsDashboard

### Out
- Pipeline visual (kanban)
- Integracao com CRM externo
- Automacao de follow-up

## Technical Notes

### Schema Changes

```sql
alter table leads add column status text
  check (status in ('novo','contatado','em_negociacao','ganho','perdido'))
  default 'novo';
alter table leads add column notes text;
alter table leads add column contacted_at timestamptz;
```
